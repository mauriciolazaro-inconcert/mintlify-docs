## Qué encontrarás en la vista detallada

- **Línea de tiempo completa** de los mensajes entre el cliente y el agente virtual.
- **Ejecuciones de herramientas** (por ejemplo, API REST, Transferencia a humano, Código).
- **Consultas a bases de conocimiento**, incluyendo resultados relevantes.
- **Aplicación de guardrails o modificadores** en cada respuesta.
- **Transferencias de agente o de canal**, cuando correspondan.
- **Variables de contexto** creadas o actualizadas en el transcurso de la conversación.

<info>
  Esta vista es ideal para **auditorías**, **análisis de calidad** y **debugging avanzado**,\
  ya que expone cada paso que se ejecutó en tiempo real.
</info>

---

## Descarga de eventos en CSV/JSON

En la parte superior de la vista detallada encontrarás el botón **Descargar CSV**.

- Al hacer clic, se descarga un **archivo CSV** que en realidad contiene un **JSON completo**\
  con todos los **eventos registrados** durante la conversación.
- Cada evento incluye:
  - **Timestamp exacto**
  - **Tipo de evento**
  - **Payload detallado** (datos enviados, respuestas de herramientas, etc.)

Esto permite **análisis offline**, integración con sistemas externos o\
trazabilidad avanzada en auditorías.

---

## Tipos de eventos conocidos

El archivo JSON puede incluir varios tipos de eventos que describen el ciclo de vida de la conversación.\
Algunos de los más comunes son:

| Evento             | Descripción                                                                  |
| ------------------ | ---------------------------------------------------------------------------- |
| `Session.destroy`  | Indica que el cliente solicitó cerrar la conversación.                       |
| `Session.start`    | Marca el inicio de una nueva interacción.                                    |
| `Message.sent`     | Mensaje enviado por el agente virtual.                                       |
| `Message.received` | Mensaje recibido del cliente.                                                |
| `Tool.executed`    | Ejecución de una herramienta (por ejemplo, API REST o Código).               |
| `Knowledge.query`  | Consulta a una base de conocimiento y recuperación de fragmentos relevantes. |
| `Transfer.done`    | Transferencia de la conversación a otro agente o equipo.                     |

<tip>
  Los nombres de eventos pueden ampliarse en el futuro;\
  revisa siempre el JSON para ver la lista completa en cada caso.
</tip>

## Buenas prácticas

- **Descarga y conserva los eventos críticos** para cumplir políticas de auditoría o compliance.
- **Filtra por tipo de evento** para depurar rápidamente (por ejemplo, solo transferencias o errores).
- **Relaciona eventos con KPIs** en [Reportes avanzados](../reporting/advanced-reports-kpis)\
  para identificar patrones de calidad o desempeño.
