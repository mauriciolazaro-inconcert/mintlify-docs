# Equipos en Inagent

Un **equipo** en Inagent es un **conjunto organizado de agentes virtuales** que colaboran para atender conversaciones en uno o varios canales.\
Dentro de cada equipo, **un agente actúa como recepcionista**: es el punto de entrada de todas las interacciones y puede transferir las conversaciones a otros agentes del mismo equipo.

---

## Estructura de un equipo

Un equipo reúne en una sola entidad:

- **Agentes virtuales**
  - Cada equipo puede tener múltiples agentes.
  - Uno de ellos es el **recepcionista**, que recibe la primera interacción y decide si continúa la conversación o la transfiere a otro agente.
- **Canales asociados**
  - Un equipo puede conectarse a **múltiples canales** (Voz, WhatsApp, Webchat, Instagram, Facebook Messenger, SMS).
  - Todos los agentes del equipo comparten estos canales y pueden responder a través de ellos según la configuración.
- **Bases de conocimiento**
  - **Del equipo**: contenido global disponible para todos los agentes.
  - **De cada agente**: información específica y exclusiva de cada uno.
  - La base de conocimiento del equipo **hereda automáticamente a sus agentes**, asegurando respuestas coherentes.
- **Herramientas**
  - **Del equipo**: integraciones y capacidades comunes (por ejemplo, API REST o código personalizado).
  - **De cada agente**: herramientas individuales que complementan o amplían las heredadas.
- **Moderadores (Guardrails)**
  - Reglas de seguridad o estilo que se aplican globalmente al equipo.
  - Cada agente puede agregar guardrails propios adicionales.
- **Tópicos (Topics)**
  - Permiten detectar temas específicos durante una conversación y **disparar automatizaciones** predefinidas.
- **Otros (timeout y reglas especiales)**
  - Configuración de **tiempo de inactividad** (timeout) y otras opciones avanzadas para manejar sesiones prolongadas o inactivas.

<Callout type="info">
Toda configuración definida a nivel de equipo (conocimiento, herramientas, guardrails) se **hereda automáticamente** a los agentes que lo conforman, evitando duplicar trabajo y garantizando coherencia.
</Callout>

---

## El rol del recepcionista

El **recepcionista** es un agente especial dentro del equipo:

- Recibe **todas las interacciones iniciales**.
- Puede **resolver directamente** la consulta o **transferir la conversación** a otro agente del equipo.
- Utiliza la configuración de **contactos** para saber a qué agentes puede derivar cada tipo de conversación.

Esta lógica permite distribuir las conversaciones según especialidades o cargas de trabajo.
